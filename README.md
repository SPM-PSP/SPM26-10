# SPM26-10 项目说明

## 项目名称
<span style="font-size:28px; color:#2b6cb0;">智教魔方</span>

## 项目介绍

本项目是一款面向高校实训教学的轻量级智能化教学辅助平台，旨在通过大语言模型与本地知识库的深度结合，解决教师在备课、考核、学情分析中的效率瓶颈，同时为学生提供实时的学习问答与练习纠错支持。

平台目前处于初创阶段，已完成总体设计，决定采用大语言模型+结合检索增强生成架构与向量数据库，在满足资源约束的前提下探索大语言模型在教育场景的落地应用。后续将围绕教学全链路持续迭代，推动教育数字化从理论走向实践。

## 行业背景和痛点分析

**行业背景**

当前，教育数字化正迈入智能化新阶段。国家大力推动“人工智能+教育”深度融合，高校实训教学作为培养学生实践能力的关键环节，亟需借助大模型技术实现转型升级。随着开源大模型的成熟，本地化、低成本的智能教学辅助系统成为可能，为解决教育资源不均、教学效率瓶颈等问题提供了技术基础。

**核心痛点**

1.  **教师备课负担重、设计难**
    -   教案设计、案例编写、考核出题需耗费大量时间查阅资料、构思内容，缺乏智能化的辅助工具。
    -   面对不同层次学生，难以快速生成差异化的教学内容，个性化教学落地困难。

2.  **学生实训缺指导、反馈慢**
    -   实操过程中遇到问题无法即时获得解答，学习进程易中断，挫败感强。
    -   练习后缺乏即时、精准的纠错反馈，错误认知得不到及时纠正，学习效果打折扣。

3.  **学情分析不精准、调整滞后**
    -   教师难以实时、量化地掌握班级整体掌握情况和学生个体的薄弱环节。
    -   教学调整往往依赖经验，缺乏数据支撑，错失最佳干预时机。

4.  **教育资源沉淀多、利用低**
    -   学校积累了大量的课件、教案、试题等资源，但以文档形式分散存储，难以被快速检索和复用。
    -   优质教学经验未能有效沉淀为可共享的知识资产。


## 核心功能

平台围绕教师、学生、管理员三类用户，构建教学全链路智能化闭环：

**1. 教师端：智能助教，提质减负**

-   **智能备课设计**：输入课程主题、难度等要求，系统自动检索本地知识库，生成结构化的教学方案，包含知识点讲解、实训案例、课堂练习及时间分配建议。
-   **智能考核生成**：选择考核范围、题型（单选、多选、编程等）、难度和题量，系统自动生成试卷及参考答案，支持导出与复用。
-   **自动化批改辅助**：对学生主观题、编程题答案进行智能分析，提供错误定位、修正建议及评分参考。
-   **学情分析看板**：可视化展示班级整体正确率趋势、知识点掌握热力图、高频错误排行榜，辅助教师精准讲评。

**2. 学生端：智能学伴，个性学习**

-   **在线学习助手（问答）**：学生输入学习疑问，系统结合本地知识库进行检索增强，生成准确、有针对性的解答。
-   **智能练习生成**：根据学生历史表现或指定知识点，动态生成个性化练习题，实现薄弱点针对性训练。
-   **即时纠错反馈**：提交练习答案后，系统自动检测错误，定位问题点，并提供详细的修正思路和示范。
-   **学习进度追踪**：记录个人练习历史、正确率变化，帮助学生了解自身学习轨迹。

**3. 管理端：智能管家，全局掌控**

-   **用户统一管理**：支持管理员对教师、学生账户进行创建、编辑、禁用及角色权限分配。
-   **课件资源管理**：按学科、类型、教师等维度分类展示所有教学资源（教案、试题等），支持预览、下载和删除。
-   **系统大屏监控**：
    -   **活跃度统计**：展示当日/本周教师、学生的使用频次和热门功能板块。
    -   **教学效率分析**：统计教师备课平均耗时、批改效率等指标。
    -   **学习效果总览**：展示全体学生平均正确率、知识点掌握薄弱排行榜，为教学管理提供决策依据。
